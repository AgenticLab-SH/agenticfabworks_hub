AgenticFabWorks는 1인 운영 웹 도구 프로젝트입니다. 도구의 실제 동작과 공개 가이드는 운영자가 검토하고, 금융·세금·건강처럼 중요한 판단에 영향을 줄 수 있는 정보는 공식 출처와 기준일을 함께 공개합니다.

Why

검색 노출보다 실제 사용을 먼저 봅니다.

같은 검색어를 겨냥한 비슷한 페이지를 대량으로 만들지 않습니다. 하나의 화면이 사용자가 하려는 일을 끝내거나, 다음 행동을 정하는 데 필요한 경우에만 공개합니다.

직접 만든 기능을 설명합니다

SKCT 가이드는 실제 서비스의 팝업, 타이머, 답안 기록과 브라우저 저장 흐름을 기준으로 작성합니다. 계산 가이드는 공개된 코드의 산식과 화면 결과를 대조합니다.

공식 관계와 전문성의 경계를 밝힙니다

SK그룹 또는 공식 SKCT 운영기관과 제휴하지 않았습니다. 세무사·노무사·의료인의 전문 상담을 제공하지 않으며, 중요한 결정은 담당 기관이나 전문가 확인을 우선하도록 안내합니다.

How

페이지를 공개하기 전 확인 순서

1. 화면의 목적을 한 문장으로 고정합니다

입력값, 사용자가 받는 결과와 그 결과로 할 수 있는 일을 적습니다. 목적이 기존 페이지와 같다면 새 페이지를 만들지 않고 기존 설명을 보완합니다.

2. 산식과 기준일을 원문에서 확인합니다

국민연금공단, 국민건강보험공단, 고용노동부, 국세청과 같은 1차 출처를 우선합니다. 적용일이 있는 값은 코드·본문·결과 안내에 같은 기준일을 사용합니다.

3. 실제 브라우저에서 결과를 재현합니다

대표 입력값과 경계값을 넣어 결과가 표시되는지 확인합니다. 데스크톱과 모바일에서 버튼, 표, 팝업, 수평 넘침과 콘솔 오류를 함께 점검합니다.

4. 제외 조건을 결과 가까이에 둡니다

세금, 수수료, 변동금리, 통상임금, 개인 건강 상태처럼 계산에 포함하지 않은 조건을 결과 화면 또는 바로 다음 설명에서 알립니다.

5. 탐색과 검색 정보를 함께 검증합니다

제목, 설명, H1, canonical, 내부 링크, 구조화 데이터와 sitemap을 확인합니다. 검색엔진에 보이는 내용과 사용자에게 보이는 내용을 다르게 제공하지 않습니다.

Corrections

오류를 발견하면 이렇게 고칩니다.

기능 오류

재현 입력값, 브라우저와 화면 상태를 확인한 뒤 코드를 수정하고 같은 조건으로 다시 검사합니다. 사용자 결과에 영향을 준 오류는 변경 기록에 원인과 수정 범위를 남깁니다.

제도·출처 변경

기준값을 갱신하고 적용일을 표시합니다. 이전 결과와 달라질 수 있으면 제목과 안내에서도 연도를 바꾸며, 근거 링크가 이동했는지 다시 확인합니다.

문장 정리와 자동화 사용

초안 정리나 반복 검사는 자동화 도구의 도움을 받을 수 있습니다. 공개 전 운영자가 기능·출처·링크와 표현을 검토하며, 자동 생성된 설명을 검증 없이 대량 게시하지 않습니다.

정정 요청

오류가 있는 URL, 입력값, 기대 결과와 실제 결과를 이메일로 보내주세요. 개인정보는 지우고 보내도 됩니다. 확인 가능한 내용은 수정한 날짜와 함께 변경 기록에 반영합니다.
